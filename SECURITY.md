# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | Yes       |

Only the latest release receives security patches.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

To report a vulnerability, use [GitHub's private vulnerability reporting](https://github.com/laplacef/deep-learning-for-cyber/security/advisories/new). You can expect an initial response within 72 hours.

Please include:
- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment

## Scope

The following are considered security issues:
- Dependency vulnerabilities in the build toolchain
- Malicious content reachable through embedded links or assets in the decks

## Out of Scope

The following are **not** security issues:
- Technical accuracy or pedagogical framing of the material
- Vulnerabilities in tools or papers referenced by the decks
